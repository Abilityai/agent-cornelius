## Core Insight

The AI Productivity Paradox: Individual developer velocity is "undeniably up" (98% more pull requests, 21% more tasks completed), but company-wide delivery metrics for throughput and quality show "no improvement whatsoever." The productivity gains simply pile up at the review queue, creating a massive bottleneck because organizations optimized only 25-35% of the workflow (coding) while ignoring the other 65-75% (review, planning, design).

---

## Context & Evidence

**The Data (Bain + Faros AI)**:
- 75% of engineers using AI coding tools
- High-AI-adoption teams: +21% tasks completed, +98% pull requests merged
- **BUT**: Company-wide throughput and quality = ZERO improvement
- Review time jumped +91% (bottleneck created)

**The Root Cause - System Blindness**:
- Writing/testing code = only 25-35% of Software Development Lifecycle (SDLC)
- Other 65-75% = planning, requirements, design, **review/validation**
- Organizations treated AI as "point solution" for coding step
- Failed to see SDLC as interconnected system
- Result: Optimized one step → Created bottleneck at next step

**The Mechanism**:
1. AI tools speed up coding (25-35% of work)
2. Faster coding → More output produced
3. More output → Hits human review queue
4. Review queue = un-optimized, human-centric, now overwhelmed
5. Review becomes bottleneck (+91% review time)
6. Productivity gains "evaporate" waiting in queue
7. Organizational throughput = unchanged

**Key Distinction (McKinsey)**:
- **95% majority**: Insert AI into old processes ("AI-Assisted" work)
- **6% High Performers**: "Fundamentally redesign workflows" ("AI-Centric" work)
